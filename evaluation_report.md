## Результати

### q001 — Щорічна відпустка після 3 місяців

**Питання:** "Я працюю в компанії 3 місяці. Чи можу вже піти у щорічну оплачувану відпустку?"

- **Контекст знайдено:** ✅ Секція "1. Щорічна відпустка"
- **Відповідь коректна:** ✅ Чітко вказано що потрібно 6 місяців безперервної роботи, а не 3
- **Fallback спрацював:** ❌ Не спрацював (правильно — контекст є)
- **Галюцинація:** ❌ Відсутня — не вигадано точних дат чи винятків
- **Confidence:** `high`

### q002 — Лікарняний без медичного документа

**Питання:** "Працівник захворів, але ще не надав медичний документ. Чи можна одразу оплатити лікарняний?"

- **Контекст знайдено:** ⚠️ Секція "2. Sick leave policy" знайдена, але Jina Reranker оцінив англомовні chunks нижче порогу 0.2
- **Відповідь коректна:** ⚠️ Fallback відповідь ("недостатньо інформації") — технічно безпечна, але не точна
- **Fallback спрацював:** ✅ Спрацював (консервативно, але з помилкою — реальна інформація є в базі)
- **Галюцинація:** ❌ Відсутня
- **Confidence:** `low`
- **Примітка:** Крос-мовний gap — BM25 не знаходить "лікарняний" → "sick leave". Jina Embeddings частково компенсує, але Reranker scored < threshold. **Слабке місце № 1.**

## Слабкі місця

1. **Cross-lingual retrieval gap** — англійські секції (2, 7, 9) знаходяться гірше при суто українських запитах через BM25 лексичний розрив. Jina Embeddings v3 частково компенсує через семантику, але не повністю.

2. **Calibration of fallback threshold** — поріг `0.2` підібраний емпірично на 5 питаннях. На більшому і різноманітнішому наборі питань може потребувати корекції — є ризик false negatives (fallback не спрацьовує коли мав би).

3. **Sentence-level chunking без overlap** — при складних питаннях що потребують контексту кількох речень поспіль, чанки без overlap можуть втрачати зв'язок між реченнями.

4. **Відсутність unit тестів** — нема автоматичних регресійних тестів для retriever та pipeline. При зміні threshold або моделі потрібен повний ручний прогін.

---

## Що покращити для production-рівня

1. **Semantic chunking з overlap** — замінити sentence-level chunks на semantic chunks з ~20% overlap для збереження контексту між фрагментами
2. **Query rewriting** — перефразовувати питання перед retrieval для покращення recall (особливо для коротких або неточних запитів)
3. **Labeled test dataset** — розширити `test_questions.json` з очікуваними відповідями для автоматичного measurement якості
4. **Fallback threshold calibration** — підібрати поріг на labeled dataset з F1 метрикою
5. **Unit tests** — покрити indexer, retriever та pipeline логіку мінімальним набором regression tests
6. **Streaming** — реалізувати `POST /ask/stream` для зниження perceived latency до <300ms TTFB
